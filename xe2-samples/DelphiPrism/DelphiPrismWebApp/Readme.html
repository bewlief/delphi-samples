Delphi Prism 2011 - World Pictures Example

ASP.NET application with AJAX and InterBase - Example

Delphi Prism provide full support for ASP.NET which has lot of cool features for .NET Web Development, this demo use DBX Framework for database access

This new version of Word Pictures explores some of these news features in Delphi Prism 2011 and .NET Framework, like:

  • Ajax Web-based Application
  • DBX Framework
  • Master Page
  • Site Maps
  • Themes

Also explains the 8 events in the global.asax file. These events are used to create application access, page access and logged users counters.

  • Application_Start
  • Session_Start
  • Application_BeginRequest
  • Application_EndRequest
  • Application_AuthenticateRequest
  • Application_Error
  • Session_End
  • Application_End

This application was tested with InterBase 2007 and 2009, and use DBX Framework.

Database Setup Information: the first time you run

This application uses an InterBase database (pictures.ib), the database scripts is located in the "database" subdirectory.  The first time you open the application the database may not be set up correctly. So it is necessary to correctly set up the connection parameters.

** Also this project include the some assemblies references which is define as C:\Program Files (x86)\*, if you use 32-bits operation system updates all reference to C:\Program Files\*

The database connection is based on dbExpress, the parameteres in Web.Config define what database to use.
The parameter DatabaseAlias define the database connection, you can connect in InterBase or other SQL database:

 For InterBase use the alias WorldPicturesInterBase

Web.config and dbxconnections.ini parameters information:

Before you run this application, please check the following web.config and dbxconnections.ini parameters for InterBase or/and other SQL database:
  • TraceTimeRequest (on / off ) (Web.config) - > if "on", displays load time at the bottom of the page.
  • Database Alias (Web.config)
    •  For InterBase use the alias WorldPicturesInterBase.
  • Use dbxconnections.ini to define the Database parameteres, you can define multiple alias on this file
    • Database --> Database parameter define the database path.
    • User_name and Password --User_name and password parameters define the user and password to access the database.
  • Database example for InterBase
    • This application uses pictures.ib database in the database sub directory

Running the Demos

  1. Start the IDE.
  2. Open the project. Complete the steps from the Setup Information section above.
  3. Compile the Applicaton (Build Menu, Build solution).
  4. Run the application with debugging (F5) or without debugging (Ctrl+F5)
